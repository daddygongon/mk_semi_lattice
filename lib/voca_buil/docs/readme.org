* name
- hc checkは英単語のetymological(語源に基づく) buildingを行うclaです．
- 単語帳のcli版です．
- dir_tree.yamlを編集しながら
  自分の記憶に残る英単語学習を目指しています．

* install
#+begin_src bash
> sudo gem install mk_semi_lattice  # install mk_sl and hc
> hc check --install                # install etymological builder yamls
> cd _stack_init_check_dirs
#+end_src
* usage
| [[file:./voca_buil/voca_buil.003.png]]
| Fig.1 hc checkの動作の概略図.
** hc check
mk_semi_latticeをgemでinstallすると，
hc(hyper cardの略)も同時にinstallされます．

: > hc check -h

を使って単語の習得checkを行います．

** tree構造
展開された_stack_init_check_dirsはtreeで確認できます．

** iterative check(#1)
_stack_init_check_dirs上で
: > hc check -i 5 -a 8 -q 5
とすると iteration=5, answer=8, quiz=5で
繰り返し(iterative) checkが走ります．

** single check(#2)
間違いの多かったdirに移動(cd)してチェックを
: > cd r1_sta
: > hc check
走らせると，r1_sta/dir_tree.yamlを参照して
単一(single)のcheckが走ります．

* dir_tree.yaml編集
| [[file:./voca_buil/voca_buil.004.png]]
| Fig.2 dir_tree.yamlのフォーマット.
Answers-Quizはdir_tree.yamlから取り出しています．
自分の進捗に合わせて編集していってください．
消してしまわずに'#'でコメントアウトするのが正しいやり方．

さらに，
: > hc stack
でバックアップをとって変えていくのがコツです．
[[../../mk_stack/docs/readme.org][stackの解説]]を参照ください．

** 基本フォーマット
YAMLのHashを真似ていますが，
実際はYAMLで読み込んでいません．
各行は，
: cap,cept=取る，捕らえる(Latin):
となっています．
先頭の空白はviewなんかを使う時の字下げになっています．

dir_tree.yamlを読み込むと，
: words[0]=['cap,cept', '取る，捕らえる(Latin)']
となります．
1. '#'以降はコメント
1. '='でseparate
1. word[0]がQuiz
1. word[-1]が Answers
1.  '/'はそのまま表示
です

これらを並べてAnswersとQuizになります．
答え合わせでは，全てを出力します．
間違った答えを確認して，覚えるkeyとしてください．

** with sample sentence
覚える単語が高度になってくると，
etymologyがよく似ているので干渉が起こりやすくなります．
そういうよく似た単語は，
sample sentence(phrase)を一緒に表示すると，
記述されている情景とともに覚えることが可能となります．
単純な繰り返しによる暗記よりも，
「周辺情報とともにバルクを増やして覚える」
暗記法のコツの一つです．

最後のAnswersに'/'でsample sentenceを書き加えるだけですが，
劇的に記憶のスピードが上がりますので試してみてください．
